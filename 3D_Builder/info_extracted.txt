No Project Chrono, a modelagem de paredes de blocos de alvenaria é geralmente abordada através do Método dos Elementos Discretos (DEM) ou de Dinâmica de Corpos Rígidos, onde cada tijolo ou bloco é tratado como um objeto individual que interage com os outros por meio de contato e fricção.
Abaixo, detalho as principais formas de implementar esses modelos:
1. Modelagem via Corpos Rígidos (Multi-body Dynamics)
Esta é a abordagem mais comum para simular o colapso de estruturas históricas ou a estabilidade de muros.
Representação: Cada bloco é um ChBody com uma geometria de colisão (ChCollisionShape).
Interação: O contato entre os blocos é gerenciado pelo sistema de colisão do Chrono, utilizando modelos de contato por força (SMC - Smooth Contact) ou por impulsos (NSC - Non-Smooth Contact).
Aplicações: Verificação de estabilidade sob gravidade, resistência a sismos e cenários de impacto.
2. Micro-modelagem vs. Macro-modelagem
No contexto do Chrono, você pode escolher o nível de detalhe:
Micro-modelagem Explícita: Você modela cada tijolo e, opcionalmente, as camadas de argamassa como elementos deformáveis ou juntas adesivas.
Micro-modelagem Simplificada: Os blocos são rígidos e a "argamassa" é representada apenas pelas propriedades de fricção e coesão nas faces de contato.
Macro-modelagem: A parede é tratada como um único bloco ou um meio contínuo (usando o módulo Chrono::FEA para elementos finitos), o que é mais eficiente para estruturas muito grandes, mas perde o detalhe da falha entre blocos.
3. Módulos Relevantes
Chrono::Multibody: O núcleo para criar os blocos e definir as juntas.
Chrono::Sensor: Útil se você estiver usando visão computacional para reconstruir paredes a partir de fotos antes de simular.
DEM-Engine: Um solver focado em GPU que permite simular milhões de blocos ou partículas granulares, ideal para simular a argamassa como um material granular ou paredes extremamente complexas.
Exemplo de Fluxo de Trabalho
Geometria: Definir o tamanho do bloco (ex: $20 \times 10 \times 10$ cm).
Loop de Construção: Usar um script (C++ ou PyChrono) para empilhar os blocos em um padrão de amarração (fio).
Propriedades de Superfície: Definir o coeficiente de fricção ($\mu$) e a coesão para simular a aderência da argamassa.
Simulação: Aplicar forças externas (vento, sismo) e observar a propagação de fissuras e o desmoronamento.
Dica Técnica: Para simulações de sismos em edifícios antigos, pesquisadores frequentemente utilizam o Chrono para realizar análise de pushover (empurrar a estrutura lateralmente até a falha) para identificar pontos críticos de vulnerabilidade.

Aqui está um exemplo básico em PyChrono para criar uma parede de blocos. Este script utiliza o sistema de contato NSC (Non-Smooth Contact), que é ideal para empilhamento de corpos rígidos onde o atrito e a gravidade são os principais fatores de estabilidade.
Exemplo de Código: Construção de Parede Simples
Este código automatiza o posicionamento dos blocos em um padrão de "amarração" (meio bloco de deslocamento por fileira).
Python
import pychrono as chrono
import pychrono.irrlicht as chronoirr
# 1. Configuração do Sistema
system = chrono.ChSystemNSC()
system.Set_G_acc(chrono.ChVectorD(0, -9.81, 0))
# 2. Material de Contato (Define Fricção e Coesão)
brick_material = chrono.ChMaterialSurfaceNSC()
brick_material.SetFriction(0.6)
brick_material.SetRestitution(0.1)
brick_material.SetCohesion(0) # Aumente para simular argamassa colante
# 3. Parâmetros dos Blocos
brick_size = chrono.ChVectorD(0.4, 0.2, 0.2) # metros
rows = 8
cols = 5
# 4. Loop para criar a parede
for r in range(rows):
    for c in range(cols):
        brick = chrono.ChBodyEasyBox(brick_size.x, brick_size.y, brick_size.z, 
                                     1000, True, True, brick_material)
        
        # Cálculo do deslocamento para o padrão de amarração
        off_x = (r % 2) * (brick_size.x / 2)
        pos_x = c * brick_size.x + off_x
        pos_y = r * brick_size.y + (brick_size.y / 2)
        
        brick.SetPos(chrono.ChVectorD(pos_x, pos_y, 0))
        system.Add(brick)
# 5. Chão estático
floor = chrono.ChBodyEasyBox(10, 0.1, 10, 1000, True, True, brick_material)
floor.SetPos(chrono.ChVectorD(0, -0.05, 0))
floor.SetBodyFixed(True)
system.Add(floor)
# 6. Visualização com Irrlicht
vis = chronoirr.ChVisualSystemIrrlicht()
vis.AttachSystem(system)
vis.SetWindowSize(1024, 768)
vis.Initialize()
vis.AddTypicalLights()
vis.AddCamera(chrono.ChVectorD(3, 3, 3))
while vis.Run():
    vis.BeginScene()
    vis.Render()
    vis.EndScene()
    system.DoStepDynamics(0.01)
Pontos Chave para Refinar seu Modelo:
Padrão de Amarração: O código acima usa (r % 2) * (brick_size.x / 2) para garantir que as juntas verticais não fiquem alinhadas, simulando uma construção real.
Simulação de Argamassa: * Para argamassa seca, use apenas Friction elevado.
Para argamassa colante, use SetCohesion(valor) no material de superfície. Isso impedirá que os blocos deslizem ou se separem até que uma força crítica seja atingida.
Estabilidade Numérica: Se a parede "vibrar" ou explodir, reduza o passo de tempo (DoStepDynamics) ou mude para o solver SMC (ChSystemSMC), que é mais estável para contatos persistentes, embora exija mais parâmetros físicos (como módulo de Young).

# Definindo a região da janela (ex: entre x=1.2 e x=2.0 e entre y=0.8 e y=1.6)
def is_inside_opening(x, y):
    # Janela
    if 1.2 < x < 2.0 and 0.8 < y < 1.6:
        return True
    # Porta (no nível do chão)
    if 3.5 < x < 4.5 and 0.0 < y < 2.0:
        return True
    return False

2. Dinâmica do Tornado com Aberturas
Com aberturas, ocorre o efeito de pressão interna. Se a janela quebrar, o vento entra no edifício, dobrando a força sobre a parede oposta. No Chrono, você pode simular isso aplicando a força de arrasto apenas nos blocos que restaram. 3. Propostas de Reforço Estrutural (Simulação no Chrono)
Como se trata de edifícios históricos, as intervenções devem ser o menos invasivas possível. Aqui estão três modalidades que você pode modelar:
A. Argamassa Técnica (Aumento de Coesão)
A forma mais simples de reforço é o preenchimento de juntas com argamassa de alta performance (base de cal hidráulica ou polímeros).
No Chrono: Aumente o parâmetro SetCohesion no ChMaterialSurfaceNSC.
Análise: Teste o valor limite onde a parede deixa de tombar como blocos soltos e passa a apresentar rachaduras diagonais (falha por cisalhamento).
B. Reforço com Fibras (TRM - Textile Reinforced Mortar)
Aplicação de uma malha de fibra de carbono ou vidro na face externa da parede. É uma técnica comum para prédios históricos.
No Chrono: Utilize o módulo Chrono::FEA. Você pode criar uma malha de elementos de casca (shell elements) "colada" à face dos blocos rígidos usando ChLinkMateFix.
Efeito: Isso impede o colapso explosivo. A malha segura os blocos no lugar mesmo após a perda de estabilidade gravitacional.
C. Tirantes ou Cabos de Protensão
Inserção de barras de aço verticais ou horizontais que atravessam a alvenaria.
No Chrono: Utilize ChElementBeamEuler (vigas) passando por dentro dos blocos ou ChLinkSpring (molas) com alta rigidez conectando o topo da parede à fundação.
Efeito: Isso aumenta a força normal sobre os blocos, o que, pela lei de Coulomb (Ff​=μ⋅N), aumenta drasticamente a resistência ao deslizamento lateral causado pelo tornado.
Sugestão de Experimento
Cenário 1 (Base): Parede com aberturas e coesão zero (apenas peso próprio).
Cenário 2 (Reforço): Mesma parede com cabos verticais (molas rígidas) simulando o travamento do telhado à fundação.

Para essa comparação, você deve modelar a fita de carbono como elementos de Casca (Shell) ou Viga (Beam) acoplados à superfície dos blocos rígidos.
Sem Reforço: Os blocos interagem apenas por contato e fricção. A falha ocorre por tombamento ou deslizamento.
Com Reforço: Os blocos são "amarrados" pela fita. A fita de carbono absorve os esforços de tração que a alvenaria não suporta.

# 1. Criar um Mesh FEA para o reforço
mesh = fea.ChMesh()
# 2. Material da Fibra de Carbono (Propriedades Típicas)
# Módulo de Young: ~230 GPa, Densidade: 1750 kg/m3

carbon_material.Set_v(0.3)

3. O que Comparar (Métricas de Análise)
Para o seu estudo de preservação, você deve focar nestes dois indicadores fornecidos pelo solver do Chrono:
A. Campo de Tensões (σ)
Sem fita: Você verá que as tensões se concentram nas quinas dos blocos (contato ponto a ponto). Quando o tornado atinge 135 mph, a tensão de tração entre os blocos supera a gravidade e a parede se desfaz.
Com fita: A fita de carbono apresentará altos níveis de Tensão de Von Mises. Ela "puxa" os blocos de volta, distribuindo a carga do vento por toda a altura da parede, evitando a abertura de juntas.
B. Deformação e Deslocamento (δ)
Meça o deslocamento do topo da parede (δtopo​).
Em prédios históricos, existe um "Estado Limite de Serviço". Mesmo que a parede não caia, se ela fletir demais, pode rachar ornamentos ou estuques originais. A fita de carbono reduz esse deslocamento drasticamente.
4. Proposta de Reforço Invisível
Para manter a estética preservada:
Aplicação Externa: A fita é aplicada e depois coberta com uma fina camada de argamassa de sacrifício compatível com a original.
NSM (Near-Surface Mounted): No Chrono, você pode simular a fita "embutida" em pequenos sulcos feitos nas juntas de assentamento.
